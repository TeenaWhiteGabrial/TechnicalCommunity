# 从原始需求到完整的前后端项目，全工作流程设计

## 概述

本流程将原始需求（几段文字描述）通过 AI 协助，转化为完整的全栈项目：
- **门户端前端**：Nuxt 3
- **管理端前端**：Vue 3
- **后端**：Node.js

### 流程总览

```
原始需求 → [阶段一] → PRD.md → [阶段二] → data-model.md + api-spec.md → [阶段三] → 代码
           需求结构化              技术设计                      代码生成
```

---

## 阶段一：需求结构化

**目标**：将原始需求转化为结构化的产品需求规格说明书（PRD）

**输入**：原始需求文本（自然语言描述）

**输出**：`PRD.md`

### 阶段一处理逻辑

1. 功能模块拆解
2. 用户角色定义
3. 核心业务流程/用例分析
4. 功能归属标注（门户端 / 管理端）

---

## 阶段二：技术设计

**目标**：将 PRD 转化为可执行的技术设计文档

**输入**：`PRD.md`

**输出**：
- `data-model.md` - 数据模型设计书
- `api-spec.md` - API 接口规格书

### 阶段二处理逻辑

1. 根据实体定义设计数据库表结构
2. 根据功能用例设计 API 接口
3. 明确门户端和管理端各自的接口范围

---

## 阶段三：代码生成

**目标**：根据技术设计文档生成完整代码

**输入**：`PRD.md` + `data-model.md` + `api-spec.md`

**输出**：
- 后端代码（Node.js）
- 门户端前端代码（Nuxt 3）
- 管理端前端代码（Vue 3）

### 阶段三处理逻辑

1. 根据 `data-model.md` 生成数据库迁移脚本和类型定义
2. 根据 `api-spec.md` 生成后端 API 路由和控制器
3. 根据 `PRD.md` 中的功能点生成前端页面组件

---

### 模板 3：api-spec.md（API 接口规格书）

```markdown
---

generatedAt: {日期}
---

# {项目名称} - API 接口规格书

## 1. 接口清单

### 1.1 门户端接口

| 接口 ID | 方法 | 路径 | 说明 | 认证 |
|--------|------|------|------|------|
| API-P-01 | GET | /api/policies | 获取政策列表 | 可选 |

### 1.2 管理端接口

| 接口 ID | 方法 | 路径 | 说明 | 认证 |
|--------|------|------|------|------|
| API-A-01 | GET | /admin/policies | 获取政策列表 | 必需 |
| API-A-02 | POST | /admin/policies | 创建政策 | 必需 |

## 2. 接口详细定义

### 接口：{接口名称}

**接口 ID**: `{api_id}`

**归属端**: portal/admin

| 配置项 | 值 |
|-------|-----|
| 方法 | GET/POST/PUT/DELETE |
| 路径 | `/api/xxx` |
| 认证 | 必需/可选/无需 |
| 权限 | {角色列表} |

#### 请求参数

| 参数名 | 位置 | 类型 | 必填 | 说明 |
|-------|------|------|------|------|
| page | query | number | 否 | 页码，默认 1 |
| limit | query | number | 否 | 每页数量，默认 10 |

#### 请求体（如适用）

```typescript
interface RequestBody {
  title: string;
  content: string;
}
```

#### 响应体

**成功响应 (200)**

```typescript
interface Response {
  code: 0;
  data: {
    list: Item[];
    total: number;
  };
  message: string;
}
```

**错误响应**

| 状态码 | 说明 |
|-------|------|
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |
